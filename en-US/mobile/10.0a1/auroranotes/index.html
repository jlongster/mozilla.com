Notes (First offered to Aurora channel users on November 11th, 2011)

Check out what’s below, and find answers to frequently asked questions. As always, you’re encouraged to tell us what you think, or file a bug in Bugzilla.

What’s New

  • NEW
    Anti-Aliasing for WebGL is now implemented (see bug 615976)
  • NEW
    Accelerated layers are now supported via OpenGL ES
  • NEW
    We've added IdexedDB APIs to more closely match the specification
  • CHANGED
    It's now even easier to set up Firefox Sync - see the "Set up Sync" link on the home tab
  • HTML5
    New <bdi> element for bi-directional text isolation, along with supporting CSS properties (see bugs 613149 and 662288)
  • FIXED
    For some users, visiting Google+ will cause the back and forward buttons to no longer work (see 670318)
  • FIXED
    Asus Transformer hardware keyboard is always en-us (see 669361)
  • FIXED
    Asus Transformer keyboard deletes two characters (see 662219)
  • FIXED
    The Swiftkey X input method can cause some character duplication issues when using "backspace" (see 672661)

Known Issues

  • Unresolved
    Searches from the URL bar do not function reliably for some locales (see 645970)
  • Unresolved
    Some recent changes in the Google layout and CSS may cause issues in non-webkit browsers like Firefox (see 668288)
  • Unresolved
    Blinking characters in form fields (see 665374)
  • Unresolved
    Unable to zoom in/out a webpage on header/footer when a double tap gesture is made (see 639504)
  • Unresolved
    Sync Now button is active even if the Sync is in progress (see 656594)
  • Unresolved
    There is an extremely rare case where the browser may become unable to load web pages or close tabs (see 676780)
  • Unresolved
    For some users HTML5 video may load slowly and display rendering artifacts on the screen during playback (see 622839)
  • Unresolved
    Reloading a page increases the zoom level (see 628269)
  • Unresolved
    Adobe Flash and other plugins are not supported. Currently, enabling the Flash plugin results in a compromised user experience across Firefox for mobile (see 687265)
