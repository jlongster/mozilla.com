How does it work?

The Mozilla Plugin Check page is a general tool for keeping web browser plugins up to date.

Fully supported browsers do the following:

  1. Gathers every plugin's metadata and optionally its live data
  2. Check with the Plugin Directory by mime-type
  3. Try to match the plugin metadata with information we know about plugins.
  4. If there is a match, then check to see if the version is outdated or has a security vulnerability.

Although the supported top plugins are a healthy portion of the traffic on the internet, we have a long way to go in terms of covering all plugins.

Plugin vendors can update their release information.

Why is IE Support Limited?

Unlike other modern browsers, Internet Explorer lacks a plugins JavaScript object. Instead of dynamically checking the plugins object to discover installed plugins, with IE plugin version checking is done with specific ActiveX code against a limited list of plugins. We'd love to check more plugins, patches welcome here.

Want to Dig Deeper?

As with all Mozilla projects, Plugin Check is developed in the open, and we welcome participation from everyone.

Hack on the project sources:

  • Pick an unsupported plugin and write JavaScript code for detecting its version on Internet Explorer using ActiveX APIs. All front end code is at the Perfidies of the Web project.
  • Hack on the back end code for plugins.mozilla.org by grabbing the code from the Plugindir project.

Chat with us on IRC (irc.mozilla.org, #pfs)

Known Issues

  • RealPlayer on Mac and Linux cannot be detected because it doesn't expose version information in the name, description, or via C++ code.
  • Adobe Reader can only be accurately detected by Firefox 3.6+. Adobe can add a version number to their Plugin Description to fix this for modern browsers.
