外掛程式更新檢查

關於我們如何進行外掛程式更新檢查，以及支援哪些瀏覽器的資訊。

完全支援的瀏覽器

瀏覽器版本所檢查的外掛程式
Apple Safari4全部
Google Chrome4全部
Mozilla Firefox3.0+全部
Opera10.5全部

部分支援的瀏覽器

瀏覽器版本所檢查的外掛程式
Microsoft Explorer7+部份*
* 因為平台的技術限制，將只能對部分的外掛程式 (BrowserPlug, Flash, Java 等) 進行版本檢查。

這是如何進行的？

Mozilla Plugin Check 是一個讓瀏覽器的外掛程式保持在最新版的通用工具。

對於完全支援的瀏覽器，將會依照下列步驟進行檢查:

  1. 收集每個外掛資訊的描述性資料 (Metadata) 與其他可選的資料
  2. 透過 MIME 類型向外掛程式目錄查詢
  3. 試著將外掛程式的描述性資料結合我們對外掛程式所瞭解的部分。
  4. 如果找的到可以對應的資料，就檢查該外掛程式是否已經過期或是安全性問題。

雖然所支援的版本是網路中比較常被使用的部分，我們仍然還有不少要努力的地方以識別所有的外掛程式。

外掛程式開發商也可以更新他們的發行資訊。

為什麼不能完全支援 IE？

不像其他新版瀏覽器，Internet Explorer 缺乏對於外掛程式的 JavaScript 物件支援。IE 只能透過一些 ActiveX 程式碼來檢查一些特定的外掛程式，而不像其他瀏覽器可以動態的檢查外掛程式物件。我們也想要持續開發以檢查更多的外掛程式，歡迎您提供軟體的補釘。

更多資訊

和全部的 Mozilla 專案一樣，Plugin Check 是開放的開發專案，我們也歡迎任何人參與。

Hack 一下程式碼:

  • 選一個未支援的外掛程式來撰寫可以透過 ActiveX API 在 Internet Explorer 上偵測版本的 JavaScript 程式碼。所有的前端程式碼都在 Perfidies of the Web 專案。
  • 到 Plugindir 專案頁面取得 plugins.mozilla.org 的後端程式碼。

已知問題

  • 無法偵測 Mac 與 Linux 版本的 RealPlayer，因為它的外掛沒有在 C++ 程式碼、軟體說明、軟體名稱當中透漏版本資訊。
  • Adobe Reader 只能在 Firefox 3.6 或以後的版本被準確地偵測到，Adobe 可以在外掛程式的說明當中增加版本號碼來解決這個問題。
